# GRPO Integration Bugs â€” 2026-03-03

## Summary

rl-values uses Tome as the inference server for GRPO training. Training produced diverging log probs (10^15 KL divergence). Root cause was multiple bugs in Tome's rollout and weight update paths.

## Bugs Fixed

### 1. Decode cache missing prefill KV (CRITICAL)

`_process_rollout_chunk` and `_process_ref_logprobs_batched` created a fresh `batched_cache` for the G-way decode phase but never populated it with the prefill KV data. The model decoded with empty context â€” logits were garbage.

**Fix** (`node.py`): After prefill, `mx.repeat` the prefill cache's `_keys`/`_values` into the batched decode cache. Pad to max prompt length when batching multiple prompts with different lengths.

### 2. Prefix cache returns uninitialized data (CRITICAL)

`prefix_cache.insert()` stored block table references after prefill, but `flush_to_pool()` was never called. The paged KV pool blocks had no data. On cache hit, `gather_kv()` read zeros/garbage.

**Fix**: 
- Rewrote `update_paged_kv` to be functional (Option A) using an optimized block-wise `__setitem__` approach.
- Updated `KVCache.gather_kv` to use the `gather_paged_kv` Metal kernel for speed.
- Added `mx.eval` to `flush_to_pool` to ensure materialization.
- Verified **3.5x to 7.2x speedup** on cache hits.

### 3. Weight update: missing LoRA scale

`UpdateWeights` computed `delta = B @ A` without the `_lora_scale` factor (e.g., 2.5 for rank=8, scale=20). The actual LoRA effect is `scale * B @ A`.

**Fix**: Added `float lora_scale = 7` to `LayerUpdate` in `inference.proto`. Scheduler passes it through. Node applies `scale * B @ A`.

### 4. Weight update: cumulative instead of replacement

`UpdateWeights` did `W += delta`, accumulating all historical deltas. Should be `W = base_W + current_delta`.

**Fix**: Use `self.reference_model` weights as the immutable base. Each update sets `policy_W = ref_W + scale * B @ A`.

### 5. gRPC message size limit

Default 4MB gRPC limit rejected LoRA weight updates (~7MB for 84 adapters).

**Fix**: Added `grpc.max_receive_message_length = 64MB` option to the gRPC server.

### 6. HTTP body size limit

Axum default 2MB body limit rejected the `/v1/weights` POST.

**Fix**: Added `DefaultBodyLimit::max(64MB)` on the `/v1/weights` route.

### P2 â€” Shared model code

Tome and rl-values have diverged copies of `model.py` / `kvcache.py` / `load_weights.py`. Both produce identical outputs (verified). Consider symlinking or making a shared package to prevent future drift.

### P2 â€” bf16 vs f32 log prob precision

Tome's `fused_log_softmax` returns bf16 (max ~0.75 diff from f32). rl-values computes log probs in f32 during the gradient pass. The gap (~0.44 per token) is within training noise but could be tightened by casting to f32 before extracting the scalar log prob in Tome.
